# hebrew-chat-digest

Робот раз в день читает твои ивритские группы (школа, дом, район), переводит на
русский и присылает короткую выжимку: что важного, какие даты и дедлайны, что
надо оплатить / принести / записаться / ответить.

Всё крутится **на твоём компьютере**. Доступ к твоему WhatsApp/Telegram никуда
не уходит — ты хостишь это сам.

---

## ⚠️ Прочитай сначала — три честных момента

- **Не совсем «бесплатно».** Сам дайджест бесплатный, но нужен свой активный
  план Claude / Cowork — робот работает там.
- **Риск бана WhatsApp.** Подключение как «связанного устройства» — это
  автоматизация, формально против правил WhatsApp, и аккаунты за это банят. Если
  номер важен — заведи отдельную SIM/eSIM под это.
- **Чужая приватность.** В дайджест попадают сообщения других родителей/соседей.
  Читать для себя — нормально. Публиковать или пересылать дайджест наружу — нет.

---

## Что понадобится

1. Компьютер, который включён по утрам (когда должен прийти дайджест).
2. Claude Code / Cowork с активным планом.
3. Телефон с твоим WhatsApp (для разового подключения по QR).

---

## Шаг 1. Поставить Cowork

Установи Claude Code / Cowork и войди в аккаунт.

## Шаг 2. Подключить источник (откуда читаем)

### Вариант A — WhatsApp (для Израиля основной)

Коннектор — open-source проект, ставится у себя:

- Репозиторий: https://github.com/lharries/whatsapp-mcp
- Устроен из двух частей: `whatsapp-bridge` (Go — подключается к твоему WhatsApp
  как связанное устройство и синкает сообщения в локальную базу) и
  `whatsapp-mcp-server` (Python — запускается через `uv`).

Следуй README проекта. Кратко:

1. Установи зависимости: **Go** и **uv** (Python-менеджер).
2. Запусти `whatsapp-bridge` — в терминале появится **QR-код**. На телефоне:
   **WhatsApp → Настройки → Связанные устройства → Привязка устройства** →
   отсканируй. Мост залогинится в твой аккаунт (первая синхронизация истории
   может занять время).
3. Пропиши MCP-сервер в своём Cowork как Local dev коннектор, команда вида:
   ```
   uv --directory <путь>/whatsapp-mcp/whatsapp-mcp-server run main.py
   ```

> Совет: боишься за основной номер — поставь WhatsApp на отдельную SIM/eSIM,
> попроси добавить этот второй номер в свои группы, и подключай по QR его.

### Вариант B — Telegram (если группы в Telegram)

Тут коннектор — отдельный open-source проект, ставится у себя:

- Репозиторий: https://github.com/chigwell/telegram-mcp
- Следуй его README: получить `api_id`/`api_hash`, авторизоваться своим
  аккаунтом, запустить мост, прописать его в `mcpServers` своего Cowork.

## Шаг 3. Выписать группы

Открой мессенджер и выпиши **точные названия** нужных групп, например:

- `כיתה ג'3 - בית ספר ...` (класс)
- `ועד בית רחוב ...` (домовой чат)

## Шаг 4. Создать рутину (расписание)

В Cowork создай **запланированную задачу** на нужное время (например, каждый
день в 08:00) с таким текстом — подставь свои группы и куда слать:

```
Прочитай сообщения за последние 24 часа из групп: «НАЗВАНИЕ_1», «НАЗВАНИЕ_2».
Переведи с иврита на русский и собери короткую выжимку:
1) Объявления и важные новости.
2) Даты, дедлайны, мероприятия — с датой и временем.
3) Что требует действия от меня: оплатить, принести, записаться, ответить.
4) Деньги: сборы, суммы, реквизиты.
Игнорируй болтовню, стикеры, оффтоп и рекламу.
Формат: по каждой группе — маркированный список, каждый пункт одной строкой.
В конце отдельный блок «Требует действия».
Если за сутки ничего важного — напиши «Ничего важного».
Пришли результат мне в WhatsApp (сообщение самому себе).
```

**Куда слать дайджест** — на выбор:

- **себе в WhatsApp** (сообщение самому себе) — проще всего, ничего больше
  настраивать не надо;
- **в Telegram** — если пользуешься (нужен Telegram-коннектор из Шага 2Б);
- **на почту**.

## Шаг 5. Проверить

Запусти рутину вручную один раз, не дожидаясь утра. Проверь: все ли группы
прочитались, адекватен ли перевод, не попал ли мусор. Подстрой промпт под себя.

---

## Как это живёт дальше

- **Компьютер должен быть включён** ко времени дайджеста (или просыпаться). На
  спящем ноуте задача не сработает.
- Раз в ~2 недели WhatsApp может попросить **переподключить устройство** — просто
  отсканируй QR заново (Шаг 2).
- Дайджест перестал приходить — почти всегда это отвалившееся подключение
  мессенджера или выключенный компьютер.
